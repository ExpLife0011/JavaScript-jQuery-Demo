preserveLastStrop Boolean 如果是true，当findBestSequence 是true的时候, 保持最后一个端点在序列中固定不变。只有findBestSequence 是true的时候可用。
restrictionAttributes String[] 用于分析限制的网络属性列表。
restrictUTurns String 指定如何处理掉头。
returnBarriers Boolean 如果是true,障碍物作为RouteTask.onSolveComplete的第二个参数返回.
returnDirections Boolean 如果是true, 在每一个结果的directions 属性中, 生成方向并返回.更多的信息可以查看Getting driving directions.
returnRoutes Boolean 如果是true,在每一个RouteResult 的route属性中,生成并返回路线。
returnStops Boolean 如果是true, 在每一个RouteResult的stops的属性中返回端点。
startTime Date 路线开始的时间. 如果没有指定,默认是在路线服务中指定的时间.
stops Object 在分析期间,加载的一组端点的网络位置。
useHierarchy Boolean 如果是true,网络的层次属性被用于分析.
useTimeWindows Boolean 如果是true,时间窗口会被用于分析.

属性详细信息

<String[]> accumulateAttributes

网络属性名称的列表将被分析累计，例如，那些应该作为响应的一部分被返回的属性。 默认值在RouteTask指定的路线网路图层中定义。用于可以指定在服务目录下"Network Dataset -> Network Attributes" as "Usage Type: esriNAUTCost"列出的任意属性名称。见Understanding the network attribute.

<Object> barriers

在分析期间, 加载的一组障碍物。 也可以是DataLayer 或者 FeatureSet 的一个实例.
参见:
doNotLocateOnRestrictedElements  

<String> directionsLanguage

.
计算方向时候使用的语言.  默认值在RouteTask指定的路线网路图层中定义。默认只有en_US,  服务器管理员可以增加另外的语言
 

<String> directionsLengthUnits

计算方向时候的长度单位。默认值在RouteTask指定的路线网路图层
已知值: esriFeet | esriKilometers | esriMeters | esriMiles | esriNauticalMiles | esriYards
参见:
DirectionsFeatureSet.totalLength

<String> directionsTimeAttribute

计算方向时，用于驾驶时间的网络属性的名称。默认定义在用于RouteTask指定的路径网络图层中。
参见:
DirectionsFeatureSet.totalTime

<Boolean> doNotLocateOnRestrictedElements

如果是true，避免通过barriers限制网络元素或因为限制条件被指定在restrictionAttributes中。
已知值: true | false
默认值: false
参见:
barriers   restrictionAttributes  

<Boolean> findBestSequence

如果是true考虑preserveFirstStop和preserveLastStop优化路线中站点的顺序。 详细信息可参考Finding the best stop sequence 和 Routing with time windows.
已知值: true | false

<Boolean> ignoreInvalidLocations

在routes中, stop没有在网络上定位,或者stop不能被得到，结果将会取决于ignoreInvalidLocations的值.
  • 当值为flase的时候, 如果没有一个特定的stop被定为为或者得到, 这个solve 操作会失败.
  • 当值为true时, 至少两个合法的stops 已经通过route被连接, 一个有效的结果被返回. 如果多个routes 在一个单一的请求中被处理, 只要一个route被建立, 一个有效的结果被返回.不能被解释的routes列表包含在RouteTask.onSolveComplete的参数消息中
已知值: true | false
参见:
RouteTask.onSolveComplete

<String> impedanceAttribute

在分析中被用作阻抗属性的网络属性名。默认值在RouteTask指定的路线网路图层中定义。 用户可以指定在服务目录下"Network Dataset -> Network Attributes" 如同 "Usage Type: esriNAUTCost"列出的任意属性名称。也可以指定'none'值来指出没有网路属性用作阻抗。如果指定一个空的数组，默认为服务的默认值。

例如，为最快路线设置impedanceAttribute="Time"和为最短的驾车时间设置impedanceAttribute="Length"，假设服务有这两个esriNAUTCost属性。

更多详细信息，见 Understanding the network attribute.

<Number> outputGeometryPrecision

泛化之后输出几何体的精确度. 如果是0,没有任何输出几何体的泛化被执行。如果当前可用,表示了MaximumAllowableOffset的参数-根据IPolycurve.Generalize执行泛化

<String> outputGeometryPrecisionUnits

输出几何体精确度的单位. 默认值是"esriUnknownUnits".

<String> outputLines

在结果中生成的输出线的类型。默认是在RouteTask中的指定的路线网络层中定义的。 可能的值, 查看NAOutputLine.

<SpatialReference> outSpatialReference

返回分析结果的几何体的空间参考的WKID,如果没有指定，几何体用地图的空间参考返回。支持的空间参考列表，见投影坐标系统和地理坐标系统。

<Boolean> preserveFirstStop

如果是true, 当findBestSequence 设置成true的时候,保持第一个端点在 固定 在序列中.  默认是在RouteTask中的指定的路线网络层中定义的.
已知值: true | false

<Boolean> preserveLastStrop

如果是true，当findBestSequence 是true的时候, 保持最后一个stop固定在序列中. 只适用findBestSequence 是true的时候. 默认是在RouteTask中的特定的路线网络层中定义的.
已知值: true | false

<String[]> restrictionAttributes

网络属性名字的列表，被用作带有分析的限制. 默认是使用在RouteTask中,定义在指定的路线网络层. 也可以指定在"Network Dataset -> Network Attributes" as "Usage Type: esriNAUTCost"下的服务目录中列出的任何属性名. 也能指定'none' 值来标识没有网络属性被用作阻抗. 如果指定一个空数组,会默认到服务的默认值。
 
参见:
doNotLocateOnRestrictedElements  

<String> restrictUTurns

详述U-Turns应该怎样处理. 默认是在RouteTask中的指定的路线网络层中定义的. 可能的值是 "esriNFSBAllowBacktrack", "esriNFSBAtDeadEndsOnly" 和 "esriNFSBNoBacktrack". 也可以查看 NAUTurn.

<Boolean> returnBarriers

如果是true，障碍物作为 RouteTask.onSolveComplete的第二个参数返回.
已知值: true | false
默认值: false
参见:
RouteTask.onSolveComplete

<Boolean> returnDirections

如果是true, 在每一个结果的directions 属性中, 生成路线并返回. 更多的信息，可以查看Getting driving directions.
已知值: true | false
默认值: false
参见:
RouteResult.directions

<Boolean> returnRoutes

如果是true，在每一个 RouteResult的route 属性中生成并返回路线
已知值: true | false
默认值: true
参见:
RouteResult.route

<Boolean> returnStops

如果是true，每一个 RouteResult的stops 属性中生成并返回端点.
已知值: true | false
默认值: false
参见:
RouteResult.stops

<Date> startTime

Route开始的时间. 如果没有指定,默认是在路线服务中指定的时间.

<Object> stops

在分析期间, 一组端点作为网络位置被装载. 可以是DataLayer 或者 FeatureSet 的一个实例. 当RouteParameters.stops 采用 FeatureSet的时候, 在FeatureSet 中的每一个要素必须有一个定义过的空间参考. 如果这个要素包含 x 和 y属性, 即使这个要素包含几何体,这些值被用作端点.  能够设置在端点上的属性列表, 查看在 Finding the best route中的"Stop properties"节.
参见:
DirectionsFeatureSet.routeName

<Boolean> useHierarchy

如果是true,网络的层次属性被用作分析. 默认是在RouteTask中的指定的路线网络层中定义的.

已知值: true | false

<Boolean> useTimeWindows

如果是true，时间窗口会被用作分析. 默认是在RouteTask中的指定的路线网络层中定义的.
已知值: true | false
